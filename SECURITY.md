# 安全策略

感谢你帮助提升 AutoGo VS Code 扩展的安全性。

## 报告方式

如果你发现了安全问题，请优先选择以下方式之一：

1. 通过 GitHub Security Advisories 私下报告
2. 如果仓库尚未启用 Security Advisories，请先联系维护者并避免公开披露细节

请尽量提供以下信息：

- 漏洞类型与影响范围
- 复现步骤或最小 PoC
- 受影响版本
- 可能的修复建议（如有）

## 处理原则

- 我们会尽快确认问题并评估影响范围
- 在修复发布前，请避免公开披露可直接利用的细节
- 修复完成后，可通过公开 Issue、PR 或更新日志补充披露

## 当前安全注意事项

- 扩展会在公共工具目录中缓存 ADB / AutoGo 工具，请确保本机权限配置符合你的安全要求
- 使用“更新 AutoGo 版本”功能前，请确认更新源为可信且可审计的公开地址
- 不要在仓库、Issue 或日志中提交密钥、令牌、测试账号或内部地址
